# Papers

## 目录目标

`docs/papers/` 用于建立“论文方法”与“仓库实现状态”之间的映射。

它应该回答：

- 这篇论文对应项目的哪一层或哪一类模型
- 当前仓库是否已经实现、部分实现，还是仅目录预留
- 这篇论文对配置、训练、评估或数据字段有什么启发

## 当前推荐内容范围

更适合写入这里的内容包括：

- `itemcf` 等已落地模型的论文背景
- 训练型候选模型的实现前置调研
- 多任务、序列推荐、生成式推荐的代表方法映射

## 页面模板建议

每篇论文页建议至少包含：

1. 论文基本信息
2. 核心问题与方法概述
3. 与 RecBench 对应的模块或模型家族
4. 当前仓库实现状态
5. 对输入字段、训练或评估的影响
6. 后续接入时应注意的工程边界

## 文件命名规范

- 使用小写
- 使用连字符
- 尽量使用稳定短名

例如：

- `itemcf-paper.md`
- `deepfm-paper.md`
- `sasrec-paper.md`

## 当前最重要的结论

`docs/papers/` 当前不应成为“未来模型名单”，而应成为“论文背景与当前实现状态之间的诚实映射层”。
